# Contributing to vibe80

First off, thank you for considering contributing to **vibe80** ❤️

We welcome contributions from the open-source community and aim to keep the process clear, fast, and respectful.

## Ways to Contribute

You can contribute by:
- Reporting bugs
- Requesting features/improvements
- Improving documentation
- Submitting code changes (vibecoded or not)
- Adding or improving tests

## Use GitHub Issues for Requests and Bugs

Please use **GitHub Issues** for:
- Bug reports
- Feature requests
- Questions about expected behavior

Before opening a new issue:
1. Search existing issues to avoid duplicates
2. Use a clear, descriptive title
3. Provide reproduction steps (for bugs)
4. Include logs/screenshots when relevant
5. Or, better, vibecode the solution using Vibe80 & push a merge request!

## Branching Model

This repository follows **GitFlow**.

Please:
- Branch from the appropriate base branch
- Keep branches focused on one change
- Rebase/update your branch before opening or updating a PR

## Commit Convention

We use **Conventional Commits**.

Examples:
- `feat: add workspace session filter`
- `fix: handle token refresh race condition`
- `docs: improve setup instructions`

## Pull Request Guidelines

To keep reviews fast and effective:
- Keep PRs small and focused
- Link related issue(s)
- Add tests when behavior changes
- Update docs when needed
- Ensure checks pass before requesting review

### Merge Policy

- Minimum: **1 reviewer approval**
- Merge method: **Squash merge**

## Quality Gates (Required)

Before opening a PR, make sure:
- Tests pass
- Lint passes

## Code of Conduct

This project adopts the **Contributor Covenant**.

Please be respectful and constructive in all interactions. If needed, maintainers may enforce the code of conduct to protect the community.

## Security Reporting

Please **do not** report security vulnerabilities in public issues.

Use private disclosure (email maintainers directly), and allow time for a fix before public disclosure.
